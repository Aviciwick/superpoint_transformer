# 🔍 batch_inference.py 错误预测点显示问题诊断与解决方案

## 📋 问题描述

使用 `batch_inference.py` 推理 ScanNet 数据集时，生成的 HTML 和 PLY 结果中**没有显示错误预测点**。

## 🔬 问题分析

### 1. 错误预测点显示的条件

根据 `src/visualization/visualization.py` 第931-944行的实现：

```python
has_error = data_0.y is not None and data_0.semantic_pred is not None
if has_error:
    # 识别错误点
    y = data_0.y
    y = y.argmax(1).numpy() if y.dim() == 2 else y.numpy()
    pred = data_0.semantic_pred
    pred = pred.argmax(1).numpy() if pred.dim() == 2 else pred.numpy()
    
    # 找出预测错误的点
    indices = np.where((pred != y) & (~np.in1d(y, ignore)))[0]
```

**关键条件：**
- ✅ 必须有 ground truth 标签 `data_0.y`
- ✅ 必须有预测标签 `data_0.semantic_pred`

### 2. 可能的问题原因

#### 原因1: `y` 标签格式问题

在数据预处理时，`GridSampling3D` transform 使用 `hist_key='y'` 参数：

```yaml
# configs/datamodule/semantic/default.yaml 第109-114行
- transform: GridSampling3D
  params:
    size: ${datamodule.voxel}
    hist_key: 'y'
    hist_size: ${eval:'${datamodule.num_classes} + 1'}
```

这会将 `y` 标签转换为**直方图格式**（2D tensor），而不是标量格式（1D tensor）。

**影响：**
- `y` 的形状从 `(N,)` 变为 `(N, num_classes+1)`
- 在可视化时，需要使用 `y.argmax(1)` 来获取最可能的类别

#### 原因2: 数据加载时 `y` 标签丢失

在 `src/datasets/base.py` 第1098-1104行：

```python
sample = NAG.load(
    self.processed_paths[idx],
    low=int(self._nano),
    keys_low=self.point_load_keys,  # 如果这里不包含 'y'，就不会加载
    keys=self.segment_load_keys,
    ...
)
```

**检查点：**
- `point_load_keys` 参数是否包含 `'y'`
- 如果 `point_load_keys=None`（默认），会加载所有键

#### 原因3: `on_device_transform` 移除了 `y` 标签

经过检查，`on_device_val_transform` 中的 transforms **不会主动移除** `y` 标签。

已在 `batch_inference.py` 的 `generate_visualization` 函数中添加调试信息：


### 方案3: 修改可视化函数以处理直方图格式的 `y`

`visualization.py` 已经正确处理了直方图格式：

```python
y = data_0.y

```

所以这部分不需要修改。

### 方案4: 检查数据预处理配置

确保数据预处理时保存了 `y` 标签：

```yaml
# configs/datamodule/semantic/scannet.yaml
# 检查是否有以下配置（如果没有，需要添加）
point_save_keys: null  # null 表示保存所有键，包括 'y'
point_no_save_keys: []  # 确保不排除 'y'
```
