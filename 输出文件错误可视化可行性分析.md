## 💡 可行性分析

### 方案1: 添加 `y` 标签到输出文件

#### ✅ 可行性：**高**

**实现步骤：**

1. **修改 `_prepare_save_data` 函数**
   ```python
   # 在第92行后添加
   y = None
   if hasattr(data, 'y') and data.y is not None:
       y = data.y.cpu().numpy()
       if y.ndim == 2:
           y = np.argmax(y, axis=1)  # 处理直方图格式
   
   # 在返回字典中添加
   return {
       'pos': pos, 'rgb': rgb, 'pred': pred, 'obj_pred': obj_pred,
       'y': y,  # 新增
       'sp_indices': sp_indices, 'has_hspt_hard': has_hspt_hard, 
       'is_hard': is_hard, 'num_points': pos.shape[0]
   }
   ```

2. **修改 `save_predictions` 函数**
   ```python
   # 在第166行后添加
   y = save_data['y']
   
   # 修改输出格式
   if y is not None:
       # 计算错误预测点
       is_error = (y != pred).astype(np.int32)
   else:
       is_error = np.zeros(num_points, dtype=np.int32)
   ```

3. **修改输出格式**
   - **TXT 格式：** 添加 `y` 和 `is_error` 列
     ```
     x y z r g b semantic_pred obj_pred y is_error sp_level_1 sp_level_2 ...
     ```
   
   - **PLY 格式：** 添加 `y` 和 `is_error` 属性
     ```
     property int y
     property int is_error
     ```

**优点：**
- ✅ 实现简单，只需修改少量代码
- ✅ 保持向后兼容（`y` 为 None 时输出 -1）
- ✅ 可以在 CloudCompare 中直接可视化

**缺点：**
- ⚠️ 增加文件大小（每点增加 2 个 int）
- ⚠️ 需要修改现有的可视化脚本

### 方案2: 仅添加错误标记列

#### ✅ 可行性：**高**

**实现步骤：**

1. **在 `_prepare_save_data` 中计算错误**
   ```python
   # 计算错误预测点
   is_error = None
   if hasattr(data, 'y') and data.y is not None:
       y = data.y.cpu().numpy()
       if y.ndim == 2:
           y = np.argmax(y, axis=1)
       pred = data.semantic_pred.cpu().numpy()
       if pred.ndim > 1:
           pred = np.argmax(pred, axis=1)
       is_error = (y != pred).astype(np.int32)
   else:
       is_error = np.zeros(num_points, dtype=np.int32)
   ```

2. **输出格式**
   - **TXT 格式：**
     ```
     x y z r g b semantic_pred obj_pred is_error sp_level_1 sp_level_2 ...
     ```
   
   - **PLY 格式：**
     ```
     property int is_error
     ```

**优点：**
- ✅ 更简洁，只增加 1 列
- ✅ 直接标记错误点，便于可视化
- ✅ CloudCompare 可以用颜色渲染 `is_error` 属性

**缺点：**
- ⚠️ 无法区分不同类型的错误（如类别混淆）
- ⚠️ 丢失了 ground truth 信息

## 🎯 推荐方案

### **方案2: 仅添加错误标记列 `is_error`**

**理由：**
1. ✅ **实现简单**：只需修改 `_prepare_save_data` 和 `save_predictions` 函数
2. ✅ **文件紧凑**：只增加 1 个 int 列
3. ✅ **易于可视化**：CloudCompare 可以直接用颜色渲染
4. ✅ **向后兼容**：当 `y` 不存在时，`is_error` 全为 0

**具体实现要点：**

```python
# 在 _prepare_save_data 函数中添加
is_error = None
if hasattr(data, 'y') and data.y is not None:
    y = data.y.cpu().numpy()
    if y.ndim == 2:
        y = np.argmax(y, axis=1)
    pred = data.semantic_pred.cpu().numpy()
    if pred.ndim > 1:
        pred = np.argmax(pred, axis=1)
    is_error = (y != pred).astype(np.int32)
else:
    is_error = np.zeros(pos.shape[0], dtype=np.int32)

# 在返回字典中添加
return {
    ..., 
    'is_error': is_error,
    ...
}
```

## 📝 注意事项

1. **Ground Truth 可用性**
   - 只有在数据集包含 ground truth 标签时才能计算错误
   - 对于测试集（无标签），`is_error` 列应为全 0

2. **标签格式处理**
   - 需要处理直方图格式的 `y` 标签（使用 `argmax`）
   - 需要处理多通道的 `semantic_pred`（使用 `argmax`）

3. **Void 类别处理**
   - 应该忽略 void 类别的错误（`y >= num_classes` 或 `y < 0`）
   - 这些点不应标记为错误

4. **文件格式兼容性**
   - 旧的脚本可能无法读取新格式
   - 建议在文件名中标注版本，如 `*_pred_v2.ply`
